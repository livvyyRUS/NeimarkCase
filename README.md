Этот проект представляет собой **комплексный фреймворк агентной системы (Agentic AI System)**. Наша система предназначена для оркестрации сложных задач путем взаимодействия между специализированными модулями-агентами и предоставлением им расширенных системных инструментов. ⚙️

Проект имитирует полный цикл разработки или анализа, где пользовательский запрос проходит через конвейер множества интеллектуальных агентов — Планировщика, Кодера, Рецензента и других узкоспециализированных модулей — которые совместно итеративно обрабатывают задачу.

## ✨ Обзор Системы (Core Concept)
Вместо прямого запроса к одной большой языковой модели (LLM), пользовательский запрос проходит через отказоустойчивый конвейер специализированных агентов. Это позволяет системе:
*   **Масштабировать сложность:** Декомпозировать крупную задачу на управляемые этапы.
*   **Повысить надежность:** Использовать критический анализ (Reviewer) и тестирование (Tester) в процессе выполнения.
*   **Выполнять действия:** Агенты не только генерируют текст, но и взаимодействуют с внешней средой через набор **инструментов (Tools)**, позволяя им выполнять реальные операции — от чтения/записи файлов до вызова команд ОС.

## 🏗️ Архитектура Проекта
Архитектура разделена на три ключевых компонента, обеспечивающие высокую модульность и расширяемость:

### 🧠 1. Модуль Агентов (`src/agents/`)
Здесь находятся классы специализированных агентов. Каждый агент наследуется от `BaseAgent` и реализует уникальную логику обработки запросов, фокусируясь на одной задаче.

| Агент | Роль | Описание |
| :--- | :--- | :--- |
| 🚀 **`OrchestratorAgent`** | Главный Координатор | Принимает запрос и управляет всем рабочим процессом: разбивает задачу, вызывает другие агенты в правильной последовательности. |
| 🎯 **`PlanAgent`** | Планирование (Planner) | Декомпозирует сложную, неструктурированную задачу на четкие, управляемые шаги. |
| 💻 **`CoderAgent`** | Кодирование (Coder) | Ответственен за генерацию, рефакторинг и написание программного кода в различных языках. |
| 🧐 **`ReviewerAgent`** | Рецензирование (Reviewer) | Критически анализирует сгенерированный код или текст, выявляя баги, уязвимости, несоответствия стилю и логические пробелы. |
| ✅ **`TesterAgent`** | Тестирование (Tester) | Автоматизирует процесс проверки: генерирует тестовые сценарии и выполняет имитации тестов. |
| 📚 `BuilderAgent`, `TechWriterAgent`| Специализированные Задачи | Построение сложных систем или написание профессиональной технической документации. |

### 🛠️ 2. Модуль Инструментов (`src/tools/`)
Этот модуль инкапсулирует взаимодействие с внешней средой (файловой системой, ОС). Он позволяет агентам вызывать системные функции как обычные методы, делая их действия прозрачными и контролируемыми.

*   **`FileTool`**: Предоставляет `@tool`-функции для файлового ввода-вывода:
    *   `read_file(path)`: Чтение содержимого указанного файла.
    *   `write_file(path, content)`: Запись или перезапись всего содержимого файла.
    *   `list_files(directory, recursive=False)`: Вывод списка файлов и папок в директории.
    *   `delete_file(file_path)`: Безопасное удаление указанного файла.

*   **`CommandTool`**: Позволяет выполнять внешние системные команды:
    *   `execute_command(command)`: Выполняет переданную команду ОС и возвращает результат выполнения.

*   **`CallAgentTool`**: Посредник для взаимодействия между агентами:
    *   `call_agent(agent, message)`: Симулирует вызов конкретного агента с передачей ему сообщения/задачи.

### ⚙️ 3. Управление Конфигурацией (`settings.py` & `main.py`)
Система использует централизованные механизмы для управления параметрами LLM и жизненным циклом приложения:
*   **`Settings`**: Класс, отвечающий за парсинг и хранение критических настроек (API ключей, URL базы, выбранная модель (`model_name`), температура).
*   **`main.py`**: Точка входа (entry point) приложения. Инициализирует `OrchestratorAgent`, загружает все необходимые настройки из файла `settings.json`, и запускает основной цикл взаимодействия с пользователем.

## 🚀 Как Начать Работу (Setup Guide)
Следуйте этим шагам, чтобы запустить и протестировать фреймворк в вашей локальной среде.

### 📂 Шаг 1: Клонирование Проекта
Клонируйте репозиторий NeimarkCase на вашу машину.

```bash
git clone https://github.com/livvyyRUS/NeimarkCase
cd NeimarkCase
```

### 📦 Шаг 2: Установка Зависимостей
Выбор менеджера зависимостей зависит от вашей среды.

**⚡ Рекомендуется (uv):**
Используйте `uv` для быстрой синхронизации и установки всех необходимых библиотек.

```bash
uv sync
```

**🐍 Альтернатива (pip/venv):**
Если вы не используете `uv`, следуйте этим шагам:

1.  Создайте виртуальное окружение:
    ```bash
    python -m venv .venv
    ```
2.  Активируйте его в вашей ОС:
    *   **Windows (CMD):** `.venv\Scripts\activate.bat`
    *   **Windows (PowerShell):** `.venv\Scripts\Activate.ps1`
    *   **macOS / Linux:** `source .venv/bin/activate`
3.  Установите зависимости:
    ```bash
    pip install -r requirements.txt 
    ```

### 🔑 Шаг 3: Конфигурация API Ключей
Создайте или обновите файл `settings.json` с вашими учетными данными и настройками LLM, используя следующий формат.

**📝 Пример структуры:**
```json
{
  "models": [
    {
      "base_url": "YOUR_API_BASE_URL", 
      "model_name": "gpt-4o", 
      "api_key": "YOUR_SECRET_API_KEY",
      "temperature": 0.2
    }
  ]
}
```

### ▶️ Шаг 4: Запуск Системы
Запустите основное приложение, которое инициирует рабочий цикл `OrchestratorAgent`.

*   **С использованием uv:**
    ```bash
    uv run main.py
    ```
*   **Без uv:**
    ```bash
    python main.py
    ```

## 🧭 Рабочий Процесс (Workflow)
Понимание, как происходит взаимодействие агентов — ключевой аспект использования фреймворка:

1.  **Ввод запроса**: Пользователь вводит задачу в командную строку (например: "Напиши простую функцию на Python, которая вычисляет факториал").
2.  **Оркестрация (`main.py`):** Запрос передается `OrchestratorAgent`.
3.  **Планирование**: Оркестратор может вызвать `PlanAgent`, который детально разложит задачу на последовательные этапы (например: Шаг 1 - Кодирование -> Шаг 2 - Тестирование типов данных -> Шаг 3 - Документация).
4.  **Выполнение:** Подзадачи направляются соответствующим агентам (`CoderAgent` пишет код, `ReviewerAgent` его проверяет и т.д.).
5.  **Интеракция с окружением:** Агенты используют инструменты (например, сохраняют результат работы в файл через `FileTool`), чтобы получить дополнительный контекст или сохранить промежуточный артефакт.
6.  **Вывод результата:** Окончательный ответ и весь ход выполненных действий логируется в консоль (`logger.info(answer)`).

## ✨ Возможности Расширения (Extensibility)
Фреймворк спроектирован с учетом максимальной гибкости, позволяя вам расширять его:

*   **➕ Добавление новых Агентов**: Создайте новый класс, наследующийся от `BaseAgent` в директории `src/agents/`, и обновите логику инициализации в `OrchestratorAgent`.
*   **🚀 Расширение Инструментов**: Внесите новую функциональность в существующие инструменты (`FileTool`) или создайте совершенно новые файлы инструментов (например, сетевые вызовы через `NetworkTool`), предоставляя агентам доступ к новым типам действий.
