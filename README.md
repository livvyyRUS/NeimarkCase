# Кейс для программы Технологии искуственного и дополненного интеллекта от Неймарка: Агентская система для разработки

Этот проект представляет собой комплексный фреймворк агентной системы (Agentic AI System). Система предназначена для оркестрации сложных задач путем взаимодействия между специализированными модулями-агентами и предоставлением агентам доступа к системным инструментам.

## 🚀 Обзор системы

Проект реализует модель работы нескольких автономных, но взаимосвязанных ИИ-агентов (Agents). Вместо прямого запроса к одной LLM, пользовательский запрос проходит через конвейер из специализированных агентов — Планировщика, Кодера, Рецензента и других модулей — которые совместно обрабатывают задачу.

Ядро системы взаимодействует с внешним окружением через набор **инструментов (Tools)**, позволяя агентам выполнять реальные операции, такие как чтение/запись файлов или выполнение команд ОС.

## ⚙️ Архитектура проекта

Архитектура разделена на ключевые компоненты, обеспечивающие модульность и расширяемость:

### 1. Модуль Агентов (`src/agents/`)
В этой директории находятся классы специализированных агентов. Каждый агент наследуется от `BaseAgent` и реализует уникальную логику обработки запросов.

*   **`OrchestratorAgent`**: Главный координатор системы. Принимает исходный запрос пользователя, разбивает его на подзадачи и направляет их по соответствующим агентам в правильном порядке.
*   **`PlanAgent`**: Отвечает за декомпозицию сложной задачи на управляемые шаги (планирование).
*   **`CoderAgent`**: Специализированный агент для генерации, рефакторинга и написания программного кода.
*   **`ReviewerAgent`**: Занимается критическим анализом сгенерированного кода или текста, выявляя потенциальные баги, уязвимости или несоответствия стилю.
*   **`TesterAgent`**: Автоматизирует процесс тестирования, может генерировать тестовые сценарии и выполнять проверки.
*   **`BuilderAgent`, `TechWriterAgent`**: Агенты для специфических задач: построение системы или написание технической документации соответственно.

### 2. Модуль Инструментов (`src/tools/`)
Этот модуль инкапсулирует взаимодействие с внешней средой (файловой системой, операционной системой). Это позволяет агентам вызывать системные функции как обычные методы, что делает их действия прозрачными и контролируемыми.

*   **`FileTool`**: Предоставляет набор `@tool`-функций для файлового ввода-вывода:
    *   `read_file(path)`: Читает содержимое указанного файла.
    *   `write_file(path, content)`: Записывает или перезаписывает файл содержимым.
    *   `list_files(directory, recursive=False)`: Выводит список файлов и папок в директории.
    *   `delete_file(file_path)`: Удаляет указанный файл (с проверками на тип объекта).
*   **`CommandTool`**: Предоставляет набор `@tool`-функций для вызова команд:
    *   `execute_command(command)`: Выполняет переданную команду ии возвращает результат
*   **`CallAgentTool`**: Предоставляет набор `@tool`-функций для вызова агентов:
    *   `call_agent(agent, message)`: Вызывает агента `agent` и передает ему на вход сообщение `message`

### 3. Управление Конфигурацией (`src/settings.py` & `main.py`)
Система использует централизованную конфигурацию для управления параметрами LLM.

*   **`Settings`**: Класс для парсинга и хранения настроек, включая параметры API ключей (`api_key`), URL базы (`base_url`), выбранную модель (`model_name`) и температуру (`temperature`).
*   **`main.py`**: Точка входа (entry point) приложения. Инициализирует `OrchestratorAgent`, загружая все необходимые настройки из файла `settings.json`, и запускает цикл взаимодействия с пользователем.

## 🧭 Использование (Workflow)

1.  **Ввод запроса:** Пользователь вводит задачу в командную строку (например: "Напиши простую функцию на Python, которая вычисляет факториал").
2.  **Оркестрация:** `main.py` передает запрос `OrchestratorAgent`.
3.  **Планирование:** Оркестратор может вызвать `PlanAgent`, который разложит задачу на этапы (Написать код -> Проверить тип данных -> Оформить документацию).
4.  **Выполнение:** Подзадачи направляются соответствующим агентам (`CoderAgent` для кода, `ReviewerAgent` для проверки и т.д.).
5.  **Интеракция с окружением:** Агенты используют инструменты (например, `FileTool`), чтобы сохранить результат работы в файл или получить дополнительный контекст из него.
6.  **Вывод результата:** Окончательный ответ и выполненные действия логируются в консоль (`logger.info(answer)`).

## 🚀 Возможности расширения (Extensibility)

Фреймворк разработан с учетом максимальной гибкости:

*   **Добавление новых Агентов**: Для создания нового агента просто добавьте новый класс, наследующийся от `BaseAgent` в `src/agents/`, и обновите логику инициализации в `OrchestratorAgent`.
*   **Расширение Инструментов**: Добавляйте новую функциональность в `FileTool` или создавайте новые файлы инструментов (например, сетевые вызовы через `NetworkTool`), предоставляя агентам доступ к новым действиям.
